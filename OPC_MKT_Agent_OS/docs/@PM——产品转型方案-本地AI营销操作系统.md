# OPC MKT Agent OS — 产品转型方案

**参与人员**: @PM（方案设计）、@Researcher（ClawX 调研 + 竞品分析）、@DEV（技术可行性评估）

## 一、转型概述

### 从哪里来
> **Web SaaS 营销工具** — 云端部署，用户通过浏览器访问，API Key 存服务端

### 到哪里去
> **本地 AI 营销操作系统** — Electron 桌面应用，用户自带 LLM Key，数据不出本地，专注出海营销场景

### 为什么要转
| 维度 | 当前 Web SaaS 模式 | 转型后桌面模式 |
|------|-------------------|---------------|
| **API 成本** | 我方承担全部 LLM 费用，烧钱 | 用户 BYO-Key，零 AI 成本 |
| **信任门槛** | 用户需把广告账号/品牌数据交给云端 | 数据全在本地，零信任顾虑 |
| **竞争壁垒** | 与 Jasper/Copy.ai 等巨头正面竞争 | 独占「本地+出海营销」空白象限 |
| **商业模式** | 月费含算力，毛利低 | 纯软件订阅/买断，毛利极高 |
| **目标用户匹配** | 出海卖家对数据安全极敏感 | 本地存储完美匹配需求 |

## 二、转型成本评估

### 2.1 现有资产盘点

我们不是从零开始。当前项目已经是一个**运行中的 MVP**：

| 资产类别 | 数量 | 详情 | Electron 可复用度 |
|----------|------|------|-------------------|
| **Web 页面** | 7 个完整页面 | Dashboard、Team Studio、Approval、Context Vault、Analytics、Publishing、CreatorFlow | **90%** — React 组件直接可用 |
| **API 路由** | 18+ 个端点 | Agent 执行、内容 CRUD、审批流、指标、设置 | **70%** — 逻辑复用，传输层改 IPC |
| **Agent 引擎** | 14 个 Agent | CEO、XHS、X/Twitter、Meta Ads、SEO、GEO、Brand Compliance 等 | **100%** — 纯 TypeScript，框架无关 |
| **数据库 Schema** | 9 张表 | 内容、指标、任务队列、品牌上下文、日程等 | **80%** — 迁移至 SQLite/本地 |
| **Skill 文件** | 14+ 个 .SKILL.md | 各 Agent 的 SOP 指令 | **100%** — 直接复用 |
| **定时调度** | node-cron 管线 | 每日 08:00-11:00 自动执行 5 步营销流水线 | **100%** — node-cron 兼容 Electron |
| **LLM 多模型支持** | 6+ Provider | Claude、OpenAI、Gemini、DeepSeek、GLM、MiniMax | **100%** — 直接复用 |
| **UI 组件库** | shadcn/ui 全套 | 14 个基础组件 + 自定义像素风组件 | **95%** — 替换 next/image 即可 |
| **数据资产** | 真实运行数据 | 品牌库 41KB、工作流历史 809KB、内容库 | **100%** — 迁移即可 |

**结论：约 85% 的代码资产可直接复用或低成本迁移。**

### 2.2 需要新增/改造的工作

| 工作项 | 工作量评估 | 复杂度 | 说明 |
|--------|-----------|--------|------|
| **Electron 壳搭建** | 2-3 天 | 低 | electron-vite + React，标准模板 |
| **IPC 通信层** | 3-5 天 | 中 | 替换 SSE/API Routes → Main↔Renderer IPC |
| **本地数据存储** | 2-3 天 | 中 | localStorage/JSON → electron-store + SQLite |
| **Keychain 集成** | 1-2 天 | 低 | 使用 keytar 存储 API Keys |
| **窗口管理** | 1-2 天 | 低 | 多窗口布局（主面板 + 聊天侧边栏） |
| **自动更新** | 1-2 天 | 低 | electron-updater，标准方案 |
| **打包分发** | 2-3 天 | 中 | macOS(.dmg) + Windows(.exe) + Linux(.AppImage) |
| **多账号管理系统** | 5-7 天 | 高 | 新功能：多平台账号绑定、切换、隔离 |
| **Skills 市场框架** | 5-7 天 | 高 | 新功能：Skill 发现、安装、热加载 |
| **平台 API 集成** | 10-15 天 | 高 | Meta Ads + X + TikTok Marketing API 真实对接 |

## 三、产品优势分析

### 3.1 竞争格局：我们处在空白象限

经调研，当前市场上**没有任何产品同时满足以下四个条件**：

```
                    通用 AI 工具                出海营销专精
                  ┌─────────────────┬──────────────────────┐
     云端 SaaS    │  Jasper ($69/月)  │  ???? (无竞品)       │
                  │  Copy.ai ($29+)  │                      │
                  │  Polsia ($49)    │                      │
                  ├─────────────────┼──────────────────────┤
     本地桌面     │  ClawX (免费)     │  ★ 我们的位置 ★      │
                  │  Accomplish      │  完全空白!            │
                  │  Manus Desktop   │                      │
                  └─────────────────┴──────────────────────┘
```

**我们要占据的位置: 「本地桌面 + 出海营销专精」— 目前零竞品。**

### 3.2 相对于 ClawX 的优势

| 维度 | ClawX | 我们 |
|------|-------|------|
| **定位** | 通用 AI 客户端（什么都能做，什么都不精） | **出海营销垂直解决方案**（深度 > 广度） |
| **开箱体验** | 用户需自己找 Skill、配 Agent | **预置 14 个营销 Agent + 完整 SOP** |
| **营销管线** | 无 | **日度自动化管线**（分析→策略→生产→审核→发布） |
| **品牌资产管理** | 无 | **Context Vault**（品牌语气、受众画像、禁用词） |
| **内容审核** | 无 | **Brand Compliance Agent**（自动品牌合规评分） |
| **数据飞轮** | 无 | **Analyst Agent**（自动提取爆款模式、优化建议） |
| **稳定性** | Gateway 崩溃是头号问题 | 我们直接调 SDK，无中间 Gateway |

**核心差异: ClawX 是「空白画布」，我们是「开箱即用的营销团队」。**

### 3.3 相对于云端 SaaS 竞品的优势

| 维度 | Jasper / Copy.ai / Polsia | 我们 |
|------|---------------------------|------|
| **价格** | $49-$69+/月（含 AI 算力） | **免费 + BYO-Key**（用户自付 AI 费用） |
| **数据安全** | 数据存云端 | **数据不出本地** |
| **定制性** | 固定功能 | **Skills 可扩展 + 开源 Agent SOP** |
| **中文支持** | 差或无 | **中文界面原生支持** |
| **出海专精** | 通用营销（美国市场为主） | **专为中国出海品牌设计** |
| **多模型** | 锁定单一 LLM | **6+ Provider 自由切换** |

### 3.4 相对于 Alibaba Accio Work 的优势

Accio Work（阿里国际）是目前最直接的威胁，但：

| 维度 | Accio Work | 我们 |
|------|------------|------|
| **生态锁定** | 绑定阿里国际站 | **平台无关**（Meta/X/TikTok/Email 全覆盖） |
| **独立站支持** | 弱（聚焦阿里平台内） | **DTC 独立站优先** |
| **透明度** | 黑箱 AI，不可控 | **本地运行，Agent SOP 可查可改** |
| **灵活性** | 阿里定义的功能 | **Skills 市场，社区驱动** |
| **定价** | 平台抽佣模式 | **一次付费或低月费** |

### 3.5 我们独有的六大壁垒

```
1. 🏗️ 已验证的 Agent 引擎
   14 个 Agent + 完整调度管线 + 真实运行数据
   竞品从零搭建需要 3-6 个月

2. 📦 Context Vault（品牌资产库）
   品牌语气、受众画像、禁用词、历史案例
   没有竞品在做这件事

3. 🔄 数据飞轮
   Analyst Agent 自动从数据中提取爆款模式
   内容越多 → 分析越准 → 下一轮内容越好
   这是真正的护城河

4. ✅ Brand Compliance（品牌合规审核）
   每条内容自动评分，确保品牌一致性
   出海品牌最怕"水土不服"的内容事故

5. 🎯 出海场景深度
   不是通用 AI 做所有事，而是深耕：
   Meta Ads 投放 → X 社交运营 → TikTok 短视频 → Email 留存
   每个环节有专属 Agent + 专属 SOP

6. 💰 BYO-Key + 本地运行
   用户成本 = 软件费(低) + AI API 费(按需)
   vs 竞品成本 = 高额月费 + 数据安全风险
```

### 与 ClawX 的关键技术差异

| 设计决策 | ClawX 做法 | 我们的做法 | 理由 |
|----------|-----------|-----------|------|
| Agent 执行 | 通过 Gateway 中间层（频繁崩溃） | **直接调 Claude Agent SDK** | 去掉 Gateway = 去掉最大故障点 |
| 数据存储 | 依赖外部数据库 | **内嵌 SQLite + electron-store** | 零配置，开箱即用 |
| Skills 加载 | 4 级目录搜索 | **本地 + 云端 Registry 双源** | 简化架构，减少冲突 |
| 进程模型 | Main + Gateway 子进程 | **Main 直接执行 Agent** | 少一个进程 = 少一个故障源 |
| 端口 | 硬编码 18789 | **无需端口**（纯 IPC） | 无端口冲突问题 |

## 七、总结

### 该不该转？

**建议转。** 理由：

1. **转型成本可控** — 85% 代码可复用，预计 6-10 周完成
2. **独占空白市场** — 「本地桌面 + 出海营销」象限零竞品
3. **商业模式更健康** — BYO-Key 模式毛利 95%，无 API 成本压力
4. **现有资产是真正壁垒** — 14 个 Agent + 数据飞轮 + 品牌合规，竞品从零追赶需半年
5. **用户需求匹配** — 出海卖家对数据安全极敏感，本地运行完美解决

### 一句话定位

> **OPC MKT Agent OS — 装在你电脑上的 AI 出海营销团队。用你自己的 AI 账号，自动管理你的所有海外平台。数据不出本地，营销永不停歇。**
