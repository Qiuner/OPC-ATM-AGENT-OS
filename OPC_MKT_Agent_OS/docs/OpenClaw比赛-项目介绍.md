## 1. 项目概述

我是一个一人公司创始人。追热点、想选题、写脚本、录播客、发公众号、做小红书……一个人要干一整支营销团队的活。内容生产全凭灵感，节奏不稳定，质量靠运气。

所以我给 OpenClaw 造了一套**营销操作系统**——ClawFlow。

它不是又一个"帮你写文案"的 AI 工具。它是一个**完整的执行平台**：OpenClaw 发出一条指令，平台就自动跑完「素材爬取 → AI 生成 → 人工审核 → 音频合成 → 飞书通知」的全链路，中间关键节点还会暂停等我确认。

**AI 干活，人类拍板。**

### 核心价值

- **平台化**：为 OpenClaw 提供 7 个标准 API 指令，覆盖内容生产全流程，即插即用
- **全自动**：一条指令触发完整工作流，无需人工逐步操作
- **人机协同**：关键节点自动暂停等待确认，内容安全可控
- **可观测**：每个工作流的每一步实时可追踪，老板看得见执行过程
- **会进化**：数据飞轮机制让系统越用越聪明，高表现内容模式自动沉淀

```

飞书消息被 `/api/openclaw/callback` 截获，解析为结构化指令，创建工作流 `wfr-mmvv652x-9c32ya`，后台开始逐步执行。

```

AI 从 5 个信息源中提炼了 3 个话题：
1. **Austin Lau 一个人做了什么** — Figma 插件把 30 分钟的广告制作压到 0.5 秒

## 3. OpenClaw 如何使用 Agent OS

### 架构关系

```
┌────────────────────────────────────────────────────────────┐
│                   OpenClaw（大脑 — 指挥官）                  │
│      理解用户意图 → 拆解任务 → 调用 Agent OS API 执行       │
└───────────────────────────┬────────────────────────────────┘
                            │ POST /api/openclaw
                            ▼
┌────────────────────────────────────────────────────────────┐
│                ClawFlow（手脚 — 执行平台）              │
│                                                            │
│  📥 素材采集     📝 脚本生成     🎙️ 播客制作               │
│  (13+平台爬虫)   (CreatorFlow)   (扣子空间TTS)             │
│                                                            │
│  📱 多平台发布   📊 数据分析     💬 飞书通知               │
│  (小红书/抖音)   (飞轮+周报)     (消息推送)                │
│                                                            │
│  👁️ 可视化看板   ✅ 人工审核     🗂️ 内容资产库            │
└────────────────────────────────────────────────────────────┘
```

OpenClaw 通过一个统一入口 `POST /api/openclaw` 发送 JSON 指令，Agent OS 自动创建工作流并逐步执行。

### 7 个标准指令

| 指令 | 功能 | 说明 |
|------|------|------|
| `collect_materials` | 素材采集 | 爬取任意 URL（公众号、小红书、Twitter、YouTube、B站、抖音等 13+ 平台） |
| `generate_script` | 脚本生成 | 调用 CreatorFlow AI，可传 URL 自动爬取后生成 |
| `generate_podcast` | 播客制作 | 生成双人对话脚本 + 触发 TTS 合成 |
| `publish_content` | 发布适配 | 将脚本适配为平台专属格式（标题/正文/标签/CTA） |
| `create_plan` | 营销计划 | 7 天内容日历 + 每日主题 + 平台分配 |
| `analyze_data` | 数据分析 | 汇总曝光、互动、线索，输出优化建议 |
| `get_status` | 系统状态 | 任务数、内容数、待审核数一览 |

### 飞书集成：自然语言也能触发

Agent OS 对接了飞书事件订阅。在飞书群发消息，自动解析为指令并执行：

```
飞书群消息：/生成播客 链接=https://xxx 主题=AI热点 时长=8min
    ↓
callback 解析为 { command: "generate_podcast", params: {...} }
    ↓
转发到 /api/openclaw → 创建工作流 → 逐步执行
    ↓
脚本完成 → 飞书推送预览 → 用户确认 → TTS → 完成通知
```

### 工作流状态机

每个工作流的生命周期：

```
queued → running → paused_for_intervention → running → completed
                         ↑
                    人工确认节点
                 （AI 不会跳过这步）
```

## 4. Agent OS 功能特性

### 4.1 万能素材采集（13+ 平台）

**目标：做 OpenClaw 的眼睛，人能看到的内容它都能抓取**

- 微信公众号、小红书、Twitter/X、YouTube、B站、抖音
- 知乎、即刻、GitHub、Reddit、LinkedIn 等
- 自动提取标题、作者、正文、图片、视频链接
- 爬取结果自动存入 Context Vault（素材资产库）

### 4.2 AI 脚本生成（多类型）

**目标：针对不同场景，生成适配平台规则的内容**

- **短视频脚本**：Hook + 痛点 + 解决方案 + CTA，适配抖音/视频号
- **播客对话脚本**：双人自然对话风格，6-8 分钟
- **小红书笔记**：标题前 18 字优化 + SEO 标签 + 互动引导
- **公众号文章**：结构化长文，引言 → 正文 → 总结 → CTA

### 4.3 播客全自动制作

**目标：展示 OpenClaw 与 Agent OS 的协作，如何创作并分发好内容**

- CreatorFlow AI 生成双人对话脚本
- 脚本推送人工确认（飞书卡片消息）
- 确认后触发扣子空间 TTS 合成音频
- 音频入库 + 保存到 CreatorFlow Episode
- 飞书通知下载链接 + 公众号文字版同步输出

### 4.4 人工审核机制（Intervention）

**目标：确保 AI 产出的内容被认可，并让 OpenClaw 在反馈中自迭代**

- 工作流在脚本生成后自动暂停
- 将脚本预览推送给用户（飞书/Web 看板）
- 用户确认后工作流自动恢复继续执行
- 支持拒绝并修改后重新生成

### 4.5 数据飞轮自动进化

**目标：每个 AI 产出的内容都可追溯，驱动好内容只需迭代**

```
内容生成 → 发布 → 回收数据 → Analyst Agent 分析 Top 20%
    → 自动更新 SKILL.md（写作规则活文档）→ 下一轮内容更好
```

小红书评分：`(收藏×3 + 点赞 + 评论×2) / 曝光量 × 100`

### 4.6 可视化工作流看板

- 每个工作流实时状态追踪（每步有时间戳 + 结果摘要）
- 审核中心、内容资产库、任务看板
- 支持 CreatorFlow 嵌入式访问

### 4.7 多 Agent 协同架构

**目标：让 OpenClaw 拥有一支 AI 营销团队，而不只是一个助手**

一个人做营销，最痛的不是写不出内容，而是**没人帮你决策、没人帮你复盘、没人帮你迭代**。所以 Agent OS 内置了一套多 Agent 协同体系——每个 Agent 专精一个领域，由 CEO Agent 统一编排调度。

#### 三个 Agent，三个角色

```
┌─────────────────────────────────────────────────────┐
│              CEO Agent（编排调度器）                    │
│   分析数据 → 制定策略 → 分配任务 → 监督执行            │
│                                                       │
│   工具：Read / Write / Glob / Grep / Bash / Agent    │
│   能力：调用 CreatorFlow API + 调度子 Agent            │
└──────────────┬───────────────────┬──────────────────┘
               │                   │
       ┌───────▼───────┐   ┌──────▼───────┐
       │  XHS Agent    │   │ Analyst Agent │
       │  小红书内容专家 │   │  数据飞轮引擎 │
       │               │   │               │
       │ • 爆款标题公式 │   │ • Top 20% 分析│
       │ • SOP 严格执行 │   │ • 胜出模式提炼│
       │ • 自检清单验证 │   │ • SKILL.md 更新│
       │ • 品牌调性守护 │   │ • 周报自动生成│
       └───────────────┘   └──────────────┘
```

#### CEO Agent：团队负责人

CEO Agent 不写内容，只做决策和调度。每日工作流：

1. **读取记忆** — 从 `memory/` 加载品牌定位、受众画像、胜出模式
2. **分析状态** — 通过 CreatorFlow API 查看竞品动态和已有素材
3. **制定计划** — 决定今日重点：创作 / 分析 / 优化
4. **派发任务** — 通过 Claude Agent SDK 的 `agents` 参数调度子 Agent
5. **汇总结果** — 写入日报，更新任务队列

```typescript
// CEO Agent 调度子 Agent 的核心代码
for await (const message of query({
  prompt: "今日营销编排...",
  options: {
    agents: {
      "xhs-agent": {
        description: "小红书营销内容创作专家",
        prompt: `${SOP} + ${胜出模式} + ${品牌调性}`,
        tools: ["Read", "Write", "Glob"],
      },
    },
    allowedTools: ["Read", "Write", "Glob", "Grep", "Bash", "Agent"],
  },
}));
```

#### XHS Agent：小红书内容创作专家

不是通用文案生成器，而是**加载了完整 SOP 的领域专家**：

- 从 `xhs.SKILL.md` 加载写作规则（标题前 18 字含 2+ 关键词、正文 300-800 字等）
- 从 `winning-patterns/` 加载 Analyst Agent 提炼的爆款模式
- 从 `brand-voice.md` 加载品牌调性，确保内容一致性
- 生成后逐项执行自检清单，不合格的重写

#### Analyst Agent：数据飞轮核心

这是整个系统的**护城河**——让 Agent OS 越用越聪明：

1. **数据收集**：读取各渠道已发布内容的表现指标
2. **特征提取**：找出 Top 20% 内容的共同特征（hook 类型、情绪触发、标题词汇、发布时间）
3. **模式对比**：与现有胜出模式对比，发现新信号
4. **更新规则**：将新发现写入 `SKILL.md`——这是活文档，不是固定模板
5. **生成周报**：简洁中文周报，发给人类决策

更新规则很克制：样本量 ≥ 10 才更新，新规则标注 `[NEW]` 和数据来源，失效规则标注 `[DEPRECATED]`，人工设定的基础规则不会被覆盖。

#### 为什么是多 Agent 而不是一个大 Agent？

| 方案 | 问题 |
|------|------|
| 一个大 Agent 全干 | Prompt 膨胀、注意力分散、创作和分析互相干扰 |
| 多 Agent 协同 | 每个 Agent 专精一域、SOP 清晰、可独立运行也可协同 |

每个子 Agent 既可以被 CEO Agent 调度，也可以独立运行（`npx tsx agents/xhs-agent.ts "主题"`），方便调试和单独使用。

## 5. 技术架构

### 技术栈

- **前端**：Next.js 15 (App Router) + TypeScript + Tailwind CSS v4 + shadcn/ui
- **Agent 引擎**：Claude Agent SDK（多 Agent 协同编排）
- **内容生成**：CreatorFlow（自研自媒体工具链，自动启动管理）
- **语音合成**：扣子空间 TTS（双人对话音频生成）
- **通知系统**：飞书机器人（事件订阅 + 消息推送 + 音频分发）
- **数据库**：Supabase PostgreSQL
- **Skills 引擎**：30+ Claude Code Skills（素材采集、图片生成、SEO 优化等）

### 核心模块

```
OPC_MKT_Agent_OS/
├── web/                          # 主 Web 平台 (Next.js 15)
│   └── src/app/api/openclaw/     # OpenClaw API Gateway（8 个端点）
│       ├── route.ts              # 主路由：指令接收 + 工作流编排
│       ├── callback/             # 飞书事件订阅回调
│       ├── execute/              # 步骤执行引擎
│       ├── intervention-callback/# 人工审核回调
│       ├── save-audio/           # 音频入库 + 飞书分发
│       ├── push-feishu/          # 飞书消息推送
│       └── workflow-status/      # 工作流状态查询
│
├── engine/                       # Agent 引擎 (Claude Agent SDK)
│   ├── agents/                   # CEO / XHS / Analyst Agent
│   ├── skills/                   # SKILL.md 活文档（Agent SOP）
│   ├── memory/                   # 品牌语气 / 受众画像 / 胜出模式
│   └── db/                       # 数据库 Schema
│
├── agent-chat/                   # Agent 对话 UI
└── docs/                         # 项目文档
```

## 7. 创新点

**大部分 AI 内容工具：** 帮你生成一段文字，然后就没了。你还得自己排版、自己发布、自己追踪效果。

**ClawFlow 不一样：**

1. **完整闭环，不是半成品** — OpenClaw 发一条指令，5 分钟后成品音频就在你飞书里。不是"帮你写个草稿"，而是从素材爬取到音频合成到通知下载的全链路交付。

2. **AI 干活，人类拍板** — Intervention 机制让工作流在关键节点自动暂停。不是"全自动发布然后翻车"，而是 AI 负责规模化，人负责品牌判断。这是真正可用于生产的设计。

3. **越用越聪明** — SKILL.md 是活文档：Analyst Agent 分析 Top 20% 的内容表现，自动更新写作规则。不是固定模板，是会进化的 SOP。

4. **252+ 次验证** — 不是 Demo，不是概念验证。这是一个正在运行的生产系统，每一次执行都有完整记录可追溯。

5. **30+ Skills 生态** — 通过 Claude Code Skills，OpenClaw 拥有了素材采集（agent-reach, 13+平台）、播客制作（ai-podcast-workflow）、文章改写（article-rewriter）、公众号发布（article-wx-publish）、图片生成（nano-banana-pro）、SEO 优化（seo-geo）等 30+ 技能。
