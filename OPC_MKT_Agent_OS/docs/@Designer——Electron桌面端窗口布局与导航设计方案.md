# Electron 桌面端窗口布局与导航设计方案

**文档类型**: UI/UX 设计方案
**完成时间**: 2026-03-26
**参与人员**: @Designer (UI 设计师)
**设计基准**: 延续 Cotify Dark Theme，对标 ClawX / Manus Desktop / Linear 桌面体验

---

## 一、设计总纲

### 1.1 设计原则

| 原则 | 说明 |
|------|------|
| **桌面原生感** | 不是"浏览器套壳"，要有原生桌面应用的质感（标题栏融合、窗口控制、系统托盘） |
| **信息密度适中** | 桌面端屏幕大，充分利用空间但不过度堆叠，保持呼吸感 |
| **延续暗色基调** | 沿用现有 Cotify Dark Theme（#030305 背景、#a78bfa 主色、#22d3ee 辅助色） |
| **键盘优先** | 核心操作支持快捷键，减少鼠标依赖 |
| **渐进式复杂度** | 首次启动极简引导，日常使用逐步暴露高级功能 |

### 1.2 设计语言延续

沿用现有 CSS 变量体系，不引入新的配色：

```
背景层级:    #030305 → #050508 → #0a0a0f → rgba(255,255,255,0.04)
主色:        #a78bfa (紫色 — 品牌/主要操作)
辅助色:      #22d3ee (青色 — 状态/实时)
警告色:      #fbbf24 (琥珀 — 待处理)
成功色:      #22c55e (绿色 — 完成/已连接)
危险色:      #ef4444 (红色 — 错误/断开)
文字层级:    #ffffff → rgba(255,255,255,0.7) → rgba(255,255,255,0.4) → rgba(255,255,255,0.25)
边框:        rgba(255,255,255,0.06) → rgba(255,255,255,0.08) → rgba(255,255,255,0.12)
```

## 五、窗口尺寸与响应式策略

### 5.1 窗口尺寸规格

| 属性 | 值 |
|------|------|
| 默认尺寸 | `1280 × 800` |
| 最小尺寸 | `960 × 600` |
| 最大尺寸 | 无限制 (跟随屏幕) |
| 记忆尺寸 | 记住用户上次关闭时的尺寸和位置 |
| 全屏支持 | macOS 原生全屏 / Windows 最大化 |

### 5.2 响应式断点策略

桌面应用不像 Web 需要 mobile/tablet 适配，但需要处理不同窗口大小：

| 断点 | 窗口宽度 | 侧边栏 | 内容区域 | 布局调整 |
|------|----------|--------|----------|----------|
| **紧凑** | 960-1080px | 强制收缩 (48px，仅图标) | 单列布局 | Dashboard 指标卡 2 列，表格隐藏低优先级列 |
| **标准** | 1080-1440px | 默认收缩 (64px)，hover 展开 | 自适应布局 | Dashboard 指标卡 4 列，正常布局 |
| **宽屏** | 1440px+ | 默认展开 (220px) | 最大 1200px 居中 + 两侧留白 | 完整布局，可考虑双栏内容 |

## 八、交互设计要点

### 8.1 窗口行为

| 交互 | 行为 |
|------|------|
| 关闭按钮 (×) | 隐藏到托盘，不退出应用（首次触发时弹出提示） |
| ⌘Q / Alt+F4 | 真正退出应用 |
| 窗口拖拽 | 标题栏区域可拖拽移动窗口 |
| 双击标题栏 | macOS: 最大化/还原; Windows: 最大化/还原 |
| 窗口 resize | 实时调整布局，侧边栏自动收缩/展开 |

### 8.2 侧边栏交互

| 交互 | 行为 |
|------|------|
| 默认态 | 收缩 64px，仅显示图标 |
| 鼠标悬停侧边栏 | 延迟 200ms 后展开至 220px（浮动覆盖，不推动内容） |
| 鼠标移出 | 延迟 300ms 后收缩回 64px |
| 点击锁定图标 | 切换固定展开模式（展开后推动内容区域） |
| `⌘\` | 切换固定展开/收缩 |

### 8.3 页面过渡动画

| 动画 | 规格 |
|------|------|
| 页面切换 | `opacity: 0→1` + `translateY: 8px→0`，200ms ease-out |
| 不使用 | 水平滑动（避免与浏览器前进/后退混淆） |
| 骨架屏 | 数据加载时显示内容骨架，`rgba(255,255,255,0.04)` 闪烁动画 |

### 8.4 通知系统

桌面端通知分两层：

**应用内 Toast (sonner)：**
- 沿用现有位置 `top-right`
- 用于操作反馈（保存成功、Agent 启动等）

**系统通知 (Electron Notification API)：**
- 当应用在后台/最小化时推送
- 触发场景: Agent 完成任务、内容审批到期、管线执行完成
- 点击通知 → 打开应用并跳转到相关页面

## 十、无障碍访问 (Accessibility)

| 要素 | 规范 |
|------|------|
| 键盘导航 | 所有交互元素可 Tab 聚焦，Enter/Space 触发 |
| Focus 样式 | `outline: 2px solid #a78bfa; outline-offset: 2px` (紫色聚焦环) |
| 对比度 | 文字/背景对比度最低 4.5:1 (AA)，主要文字 `#ffffff` on `#030305` = 19.7:1 |
| ARIA 标签 | 导航区 `role="navigation" aria-label="主导航"`，主内容区 `role="main"` |
| 屏幕阅读器 | 页面切换时 `aria-live="polite"` 播报当前页面名称 |
| 动画开关 | 尊重 `prefers-reduced-motion`，禁用环境光晕/过渡动画 |
