# X/Twitter 创作 Agent SOP

## 启动前必读
1. 读取 `./memory/context/brand-voice.md`（品牌调性）
2. 读取 `./memory/context/target-audience.md`（目标受众）
3. 确认内容类型（单条推文 / Thread / 引用推文）

## 内容类型

| 类型 | 适用场景 | 字数限制 | 特点 |
|------|---------|----------|------|
| 单条推文 | 观点/金句/热点评论 | 280 字符 | 精炼有力，一击即中 |
| Thread | 深度分析/教程/故事 | 5-15 条 | 首条是 Hook，逐步深入 |
| 引用推文 | 回应行业热点/互动 | 280 字符 | 补充独特视角 |

## 推文撰写规则

### 1. 单条推文公式
- 观点型: [鲜明观点] + [一句话理由] + [反直觉元素]
- 数据型: [具体数字] + [背后洞察] + [行动启示]
- 金句型: [类比/隐喻] + [核心主张]
- 问题型: [引发思考的问题] + [自己的回答方向]

### 2. Thread 结构
- 第 1 条: Hook — 用最强观点或数据开头，引发好奇
- 第 2-N 条: 正文 — 每条一个核心观点，独立可读
- 最后 1 条: CTA — 引导关注/转发/讨论
- 每条之间有逻辑递进关系
- 适当使用 emoji 增强可读性（不超过 2 个/条）

### 3. 语言风格
- 英文为主（国际受众），中文版单独生成
- 短句优先，避免从句嵌套
- 使用 "you" 直接对话感
- 避免 "I think"，改用 "Here's the thing:"
- 专业术语出现后用大白话解释

### 4. 互动优化
- 推文末尾用问题引发评论
- Thread 中间穿插互动点（"Agree or disagree?"）
- 提及相关账号增加曝光（@mention）
- 使用 2-3 个相关 hashtag

```markdown
[推文内容]

### 中文版
[推文内容]

## Thread（如适用）

[内容]

### Tweet 2

```

- [ ] 有明确 CTA

- [ ] 语言风格符合 X 平台调性
- [ ] 内容与品牌调性一致
- [ ] 有互动引导（提问/投票/观点碰撞）
