# 播客制作 Agent SOP

## 启动前必读
1. 读取 `./memory/context/brand-voice.md`（品牌调性）
2. 读取 `./memory/context/target-audience.md`（目标受众）
3. 确认播客类型（独白 / 对谈 / 圆桌）

## 播客类型

| 类型 | 适用场景 | 时长 | 特点 |
|------|---------|------|------|
| 独白式 | 品牌故事/个人观点/干货分享 | 5-15 分钟 | 一人深度讲述，节奏紧凑 |
| 对谈式 | 产品对比/案例拆解/用户访谈 | 10-25 分钟 | 两人自然对话，有问有答 |
| 圆桌式 | 行业趋势/热点讨论/多角度分析 | 15-30 分钟 | 多人讨论，观点碰撞 |

## 脚本结构

### 1. 开场（30-60秒）
- Hook: 用一个引人入胜的问题、数据或故事开场
- 自我介绍: 简短，不超过 2 句
- 本期预告: 告诉听众今天能学到什么（1-2 句）

### 2. 正文（按段落组织）
- 每个段落围绕一个核心观点
- 段落之间有自然过渡语（"说到这个..."、"另一个有趣的点是..."）
- 每 3-5 分钟插入一个故事/案例/数据，保持吸引力
- 对谈式：保持对话自然，避免念稿感

### 3. 金句（每期至少 2 句）
- 可被截取为短视频/图文引用的精华句
- 观点鲜明、简短有力、有记忆点
- 示例："AI 不会取代你，但会用 AI 的人会取代你"

### 4. 结尾（30-60秒）
- 总结本期核心观点（不超过 3 点）
- CTA: 引导关注/订阅/评论
- 预告下期内容（如有）

## 输出格式

```markdown
# 播客脚本: [标题]

## 元数据
- 类型: [独白/对谈/圆桌]
- 预计时长: [X 分钟]
- 目标受众: [描述]
- 关键词: [用于 SEO 和平台标签]

## 开场
[开场脚本，标注语气和停顿]

## 段落 1: [小标题]
[正文脚本]
> 金句: [可截取的精华句]

## 段落 2: [小标题]
[正文脚本]

## 结尾
[结尾脚本，含 CTA]

## 配套内容建议
- 标题选项: [2-3 个标题]
- 封面文案: [1 句话]
- 社交媒体推广文案: [适合小红书/X 的简短推广文]
```

## 语言风格
- 口语化，像在和朋友聊天
- 避免书面语和官方腔调
- 适当使用"嗯"、"对"、"你知道吗"等口语词增加自然感
- 专业术语出现后立即用大白话解释

## 自检清单

- [ ] 有明确 CTA

- [ ] 预计时长在目标范围内
- [ ] 内容与品牌调性一致
