# 数据分析 Agent SOP

## 启动前必读
1. 读取 ./memory/winning-patterns/*.md（当前胜出模式）
2. 读取 ./skills/*/SKILL.md（当前各渠道 SOP）
3. 获取内容表现数据（从 CreatorFlow API 或数据库）

## 分析流程

### Step 1: 数据采集
- 调用 CreatorFlow `GET /api/publish` 获取已发布内容
- 调用 `POST /api/my-account` 同步小红书账号数据
- 调用 `POST /api/review/sync-xhs` 同步复盘数据

### Step 2: 评分计算
按渠道使用不同公式：

**小红书**
```
score = (收藏×3 + 点赞 + 评论×2) / 曝光量 × 100
```

**抖音**
```
score = 完播率×50 + (点赞+分享×3) / 播放量 × 50
```

**X/Twitter**
```
score = (likes + retweets×3 + replies×2) / impressions × 100
```

### Step 3: 排序分层
- Top 20%：标记为「胜出内容」，进入模式提取
- Middle 60%：标记为「合格」，不做特殊处理
- Bottom 20%：标记为「待优化」，分析失败原因

### Step 4: 模式提取（从 Top 20%）
从胜出内容中提取以下维度的规律：

| 维度 | 提取内容 | 示例 |
|------|---------|------|
| 标题模式 | 哪种标题结构表现好 | 数字型 > 痛点型 |
| 内容结构 | 段落数、字数范围 | 5段，400-600字最佳 |
| Hook 类型 | 哪种开头吸引人 | 反常识 > 个人故事 |
| 话题标签 | 哪些标签带来流量 | #AI工具# 曝光最高 |
| 发布时间 | 什么时间发布效果好 | 周二/四 20:00-22:00 |
| 情感调性 | 积极/中性/挑战 | 「挑战常识」类最高 |
| CTA 方式 | 哪种引导效果好 | 提问式 > 收藏式 |
| 内容长度 | 最佳字数区间 | 500-650字互动率最高 |

置信度判定：
- 高置信：≥5 条样本支持
- 中置信：3-4 条样本
- 低置信：<3 条，标注为「待验证」

### Step 5: 更新 winning-patterns
- 文件路径：`./memory/winning-patterns/{channel}-patterns.md`
- 新发现的模式追加到文件
- 旧模式如果近 30 天无新数据支持，降低权重
- 每条模式标注：置信度 + 样本数 + 最后验证日期

### Step 6: 生成报告
输出周报到 `./memory/reports/weekly-{date}.md`

## 报告模板

```markdown
# 周报 {date}

## 数据概览
| 渠道 | 发布数 | 平均 score | Top 内容 | 趋势 |
|------|--------|-----------|---------|------|

## Top 3 胜出内容
1. 「标题」— score: X, 原因分析
2. ...

## 新发现模式
- [模式描述] — 置信度: 高/中/低, 样本数: N

## Bottom 3 待优化
1. 「标题」— score: X, 优化建议
2. ...

## 下周建议
- 多做: [胜出模式]
- 少做: [失败模式]
- 测试: [待验证假设]
```
