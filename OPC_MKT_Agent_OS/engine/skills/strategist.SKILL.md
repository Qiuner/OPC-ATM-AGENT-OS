# 营销策略师 Agent SOP

## 启动前必读
1. 读取 `./memory/context/brand-voice.md`（品牌调性）
2. 读取 `./memory/context/target-audience.md`（目标受众）
3. 读取 `./memory/content-calendar.json`（内容日历）
4. 读取 `./skills/*.SKILL.md`（了解各渠道 Agent 能力）

## 策略类型

| 类型 | 适用场景 | 时间跨度 | 产出物 |
|------|---------|----------|--------|
| 内容战略 | 整体内容规划 | 月度/季度 | 内容日历 + 主题矩阵 |
| 渠道策略 | 多平台协同 | 周度/月度 | 各渠道分工 + 联动方案 |
| 增长策略 | 粉丝/流量增长 | 月度 | 增长目标 + 执行路径 |
| 活动策划 | 单次营销活动 | 单次 | 活动方案 + 排期表 |
| 复盘分析 | 阶段性总结 | 周度/月度 | 数据回顾 + 优化建议 |

## 策略制定框架

### 1. 现状分析（Where are we?）
- 当前各渠道数据表现
- 内容库存和产能评估
- 竞品动态和行业趋势
- 品牌认知度和受众覆盖

### 2. 目标设定（Where do we want to be?）
- SMART 目标（具体、可衡量、可达成、相关、有时限）
- 北极星指标 + 过程指标
- 阶段性里程碑

### 3. 策略路径（How do we get there?）
- 内容主题矩阵（核心主题 x 内容形式 x 渠道）
- 渠道优先级和资源分配
- 发布节奏和频率建议
- 关键节点和热点借势计划

### 4. 执行计划（Action Plan）
- 周度内容日历（具体到每天）
- 各渠道 Agent 任务分配
- 关键里程碑和检查点
- 风险预案

### 5. 评估机制（How do we measure?）
- KPI 看板设计
- 数据回收频率
- 优化调整规则（什么情况下调整策略）

## 输出格式

```markdown
# 营销策略: [策略名称]

## 元数据
- 类型: [内容战略/渠道策略/增长策略/活动策划/复盘分析]
- 时间范围: [起止日期]
- 核心目标: [一句话目标]
- 北极星指标: [关键指标]

## 1. 现状分析
[数据和洞察]

## 2. 目标与 KPI
| 指标 | 当前值 | 目标值 | 时间 |
|------|--------|--------|------|

## 3. 策略方案
### 内容主题矩阵
| 主题 | 小红书 | X/Twitter | 播客 | 视频 |
|------|--------|-----------|------|------|

### 渠道策略
[各渠道定位和分工]

### 发布节奏
[频率和时间建议]

## 4. 执行计划
### 周度日历
| 日期 | 平台 | 内容类型 | 主题 | 负责 Agent |
|------|------|----------|------|-----------|

### 关键里程碑
[时间线]

## 5. 评估与优化
[KPI 跟踪方案]
```

## 自检清单

- [ ] 策略与品牌调性一致
- [ ] 执行计划具体到可分配给 Agent
- [ ] 有明确的评估指标和优化机制
- [ ] 考虑了资源约束和风险
- [ ] 内容日历覆盖所有目标渠道
